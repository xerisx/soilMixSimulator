# Project Guidance

## 基本方針
- まず現状コードを確認してから対応すること。
- 実装は常に最小変更を優先すること。
- 大きなリファクタ、不要な構造変更、責務の再分割は行わないこと。
- 既存のデザインテイスト、コンポーネント、スタイルを極力維持・再利用すること。
- 変更点の報告は簡潔に行うこと。

## このプロジェクトの目的
このプロジェクトは、ブラウザ上で動作する用土配合シミュレータである。

目的:
- 配合による違いを視覚的・直感的に理解できるようにする
- 排水性 / 保水性 / 通気性 / 保肥力などを比較理解できるようにする
- 触って楽しい UI にする

重視すること:
- 完全な物理再現ではなく、「それっぽく正しい」こと
- 専門家向けの厳密性より、「見て分かる」こと
- UI/UX の気持ちよさ、一貫性、分かりやすさ

## 実装判断の優先順位
判断に迷った場合は、以下を優先すること。

1. 視認性
2. 情報の優先順位の明確さ
3. 操作の分かりやすさ
4. レイアウトの安定性
5. 既存 UI との整合性
6. 厳密さより直感性

## UI改善の基本ルール
- 新機能追加よりも、視認性・情報の強弱・操作の気持ちよさ・レイアウト安定性を優先すること。
- まずは既存構造を壊さず、余白・階層・密度・配置の微調整で改善できるかを検討すること。
- コントラスト不足は避けること。淡色要素でも読めることを優先すること。
- 一覧 UI では、1項目ごとの情報密度と比較しやすさを重視すること。
- 可変長テキストによって、ボタン位置や主操作の位置が揺れないようにすること。
- 主操作と補助情報は明確に分離すること。

## シミュレータUIに関する前提
- 3カラム構成:
  - 左: 操作エリア
  - 中央: 鉢シミュレーション
  - 右: 配合分析
- 「鉢に対する操作」は鉢の下に置く方針を基本とする。
- 主 CTA は明確にし、補助操作は主張を弱めること。
- ボタンや説明を鉢内部、鉢線、説明カードの上に不自然に重ねないこと。
- サイズ UI は、操作部と説明部を分離し、説明文の長さでボタン位置が動かないようにすること。

## 資材カードUIに関する前提
- サイズ選択は主操作として扱うこと。
- サイズ説明はサイズUIの近くにまとめること。
- 「有機」「排水」「通気」などのタグは補助情報として扱うこと。
- カード縦幅は必要以上に大きくしないこと。
- 情報量を減らさずに、再配置で一覧性を上げることを優先すること。

## 分析UIに関する前提
- 右カラムの分析は、このツールの価値が集まる重要領域として扱うこと。
- 情報量が増えても見切れないようにすること。
- 必要に応じて、右カラムまたは分析領域に内部スクロールを持たせること。
- 開閉 UI は、見出しと開閉対象が一体に見えることを優先すること。
- アコーディオンは、開いたときに 1 つのカードとして自然につながって見えること。

## 物理・分析ロジックに関する前提
- 完全な物理再現は目指さない。
- 数値や状態ラベルは、厳密な科学値よりも「見た目と説明が矛盾しないこと」を優先する。
- 「どれだけ入ったか」と「粒子間にどれだけ隙間があるか」は別概念として扱う。
- 空隙率や通気的な指標は、必要に応じて以下の要素を使って擬似的に推定してよい:
  - porosity
  - coarseRatio
  - compressibility
  - 粒径
  - 鉢サイズ
- 同じ資材でも、鉢サイズや粒径との比率で見え方や詰まり方が変わることを考慮してよい。
- ただし、実装は過剰に複雑にせず、「傾向として自然」な範囲に留めること。

## 作業時の出力ルール
- まず何を確認したかを簡潔に整理する
- 次に、最小変更でどう直したかを簡潔に述べる
- 不要に長い説明は避ける
- 大きな変更が必要そうでも、先に最小変更案を検討する
- 変更理由は、視認性・情報設計・操作性の観点で説明する

## 禁止寄りのこと
- 頼まれていない大規模リファクタ
- 既存 UI の雰囲気を壊す全面刷新
- ロジックの厳密性を優先しすぎて UX を悪化させること
- 可変長文言のせいでレイアウトが不安定になる実装
- 変更点の説明なしに広範囲を触ること
